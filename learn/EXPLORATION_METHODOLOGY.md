# Claude Code 源码探索方法论

## 核心原则

**深度优先于广度** - 每个模块都要彻底理解，而不是浮于表面

## 探索流程 (每个模块)

### 1. 入口点追踪
- 找到模块的入口文件和导出
- 理解模块在整个系统中的位置
- 画出调用链

### 2. 数据结构分析
- 定义的数据类型/接口
- 数据流动方向
- 状态管理方式

### 3. 核心逻辑剖析
- 主要函数的职责
- 分支/条件判断的逻辑
- 边界情况处理

### 4. 与其他模块的交互
- 依赖哪些模块
- 被哪些模块依赖
- 接口契约

### 5. 设计模式提取
- 使用的设计模式
- 设计决策的原因
- 潜在的扩展点

### 6. 流程图/时序图
- 关键流程的可视化
- 状态转换

## 每个功能实现前必须回答的问题

1. **这个功能在 Claude Code 中叫什么？**
2. **它的完整流程是什么？**
3. **有哪些关键的分支/状态？**
4. **并行/串行的决策点在哪里？**
5. **边界情况如何处理？**
6. **和其他功能的交互点是什么？**

## 文档要求

每个模块探索后必须产出：
- `learn/topics/[模块名].md` - 至少 200 行深度分析
- 包含代码引用 (文件:行号)
- 包含流程图或时序图
- 包含设计决策说明

## 验证方式

实现完成后，必须：
1. 在文档中找到对应的设计点
2. 确认实现符合源码设计
3. 标注差异点（如有）
